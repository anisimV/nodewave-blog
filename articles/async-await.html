Async

Как работает async/await

Разбираем асинхронность в JavaScript простыми словами: что такое Promise, зачем нужны async/await и как писать более понятный асинхронный код.

Почему вообще нужна асинхронность

В JavaScript часто бывают задачи, которые не выполняются мгновенно. Например:

  • запрос на сервер
  • чтение файла
  • ожидание ответа от базы данных
  • таймер

Если бы программа полностью останавливалась и ждала завершения каждой такой операции, приложение работало бы медленно и неудобно.

Поэтому в JavaScript есть асинхронность — механизм, который позволяет запускать долгую операцию и не блокировать выполнение остального кода.

Синхронный код

Синхронный код выполняется строго по порядку: строка за строкой. 

console.log('Шаг 1');
console.log('Шаг 2');
console.log('Шаг 3');

Результат будет таким:

Шаг 1
Шаг 2
Шаг 3

Тут всё просто: каждая следующая команда начинается только после завершения предыдущей.

Асинхронный пример с setTimeout

Теперь посмотрим на простой асинхронный пример: 

console.log('Начало');

setTimeout(() => {
  console.log('Сообщение с задержкой');
}, 2000);

console.log('Конец');

Многие новички ожидают такой порядок:

Начало
Сообщение с задержкой
Конец

Но на самом деле будет так:

Начало
Конец
Сообщение с задержкой

Почему? Потому что `setTimeout()` запускает асинхронную задачу. JavaScript не ждёт 2 секунды на этой строке, а идёт дальше, выполняя остальной код.

Что такое Promise

Promise — это объект, который представляет результат асинхронной операции.

Если говорить очень просто, Promise — это “обещание”, что результат появится позже:

  • либо успешно
  • либо с ошибкой
const promise = new Promise((resolve, reject) => {
  const success = true;

  if (success) {
    resolve('Операция выполнена');
  } else {
    reject('Произошла ошибка');
  }
});

Здесь:

  • `resolve()` означает успешное завершение
  • `reject()` означает ошибку

Как работать с Promise через then/catch

До `async/await` с Promise обычно работали так: 

const getData = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve('Данные получены');
    }, 1500);
  });
};

getData()
  .then((result) => {
    console.log(result);
  })
  .catch((error) => {
    console.log(error);
  });

Этот вариант рабочий, но когда асинхронных шагов становится много, код начинает становиться менее удобным для чтения.

Что делает async

Ключевое слово `async` ставится перед функцией. Оно означает, что функция всегда возвращает Promise. 

async function sayHello() {
  return 'Привет';
}

Хотя кажется, что функция просто возвращает строку, на самом деле JavaScript оборачивает результат в Promise.

Что делает await

`await` можно использовать только внутри `async`-функции. Он говорит: “подожди результат Promise, а потом продолжай”. 

const getData = () => {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve('Данные загружены');
    }, 1500);
  });
};

async function showData() {
  const result = await getData();
  console.log(result);
}

showData();

Здесь происходит следующее:

  1. вызывается `showData()`
  2. внутри вызывается `getData()`
  3. `await` ждёт, пока Promise завершится
  4. после этого результат записывается в `result`
  5. значение выводится в консоль

Чем async/await удобнее

Главное преимущество в том, что код выглядит почти как обычный последовательный код.

Через `then`:

getUser()
  .then((user) => {
    return getPosts(user.id);
  })
  .then((posts) => {
    console.log(posts);
  })
  .catch((error) => {
    console.log(error);
  });

Через `async/await`:

async function showPosts() {
  try {
    const user = await getUser();
    const posts = await getPosts(user.id);

    console.log(posts);
  } catch (error) {
    console.log(error);
  }
}

Второй вариант обычно легче читать, особенно если шагов несколько.

Как обрабатывать ошибки

При работе с `async/await` ошибки обычно обрабатывают через `try...catch`. 

const getUser = () => {
  return new Promise((resolve, reject) => {
    const hasError = false;

    if (hasError) {
      reject('Не удалось получить пользователя');
    } else {
      resolve({ id: 1, name: 'Anisim' });
    }
  });
};

async function loadUser() {
  try {
    const user = await getUser();
    console.log(user);
  } catch (error) {
    console.log('Ошибка:', error);
  }
}

loadUser();

Если Promise завершится с ошибкой, управление перейдёт в блок `catch`.

Пример из реальной логики

Представим, что мы сначала загружаем пользователя, а потом по его `id` загружаем список задач. 

const getUser = () => {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve({ id: 7, name: 'Anisim' });
    }, 1000);
  });
};

const getTasks = (userId) => {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve([
        { id: 1, title: 'Изучить async/await' },
        { id: 2, title: 'Сделать mini project' }
      ]);
    }, 1000);
  });
};

async function showTasks() {
  try {
    const user = await getUser();
    const tasks = await getTasks(user.id);

    console.log('Пользователь:', user.name);
    console.log('Задачи:', tasks);
  } catch (error) {
    console.log('Ошибка:', error);
  }
}

showTasks();

Такой код читается намного проще, чем длинная цепочка `.then()`.

Частые ошибки новичков

  • используют `await` вне `async`-функции
  • забывают обрабатывать ошибки
  • думают, что `async` делает код полностью синхронным
  • не понимают, что `async`-функция возвращает Promise

Например, вот так делать нельзя:

function loadData() {
  const result = await getData();
  console.log(result);
}

Здесь будет ошибка, потому что `await` можно использовать только внутри `async`-функции.

Итог

`async/await` — это удобный способ работать с асинхронным кодом в JavaScript.

Он не отменяет Promise, а просто даёт более понятный синтаксис для работы с ними.

Если ты понимаешь:

  • что асинхронные операции завершаются не сразу
  • что Promise хранит будущий результат
  • что `await` ждёт завершения Promise
  • что ошибки обрабатываются через `try...catch`

значит ты уже хорошо понимаешь основу темы.
